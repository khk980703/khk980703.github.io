Problem Solving



Time I was blocked on a simple problem.

When I was trying to utilize the JavaScript builtin function 'map()', I struggled with an error corresponding to that. Eventually, I found out that for some reason I thought the function changes the original array, but actually, it returns a new array with the changed variables. All I needed to do was to assign a new variable name to it to solve the problem.

Throughout most of my errors, I've tried the rubber ducky method, reading error messages, and console.logging the most. Usually, with just those three methods, I was able to get through most of the simple problems.

Throughout these trials and errors, I've realized that most of the time there are just a few pieces of the puzzle missing. Having a thorough look into what I've written and drawing it out in my head gives me a clear outcome of my code.



Time I elegantly solved a problem.

From one of the stretch exercises, I struggled to understand the requirements of the question. It was asked to assign a value as a function with the key 'win'. At first, I did not realize we could assign a function itself as a value in an object.

I decided to look into the code with the problem-solving methods that I listed above and they helped a lot. Looking into the code from the start testing out each step that I took revealed where I've gone wrong and was able to fix the problem.



Reflect on how confident you feel using each of these problem-solving techniques/processes.

  • Pseudocode - I have not used too much pseudocode because, during the kata exercise, there were instructions commented clearly already. In the future, I will definitely try pseudo-coding.

  • Trying something - I have never given up on a problem so far meaning I am confident with the skill, of trying something.

  • Rubber ducky method - This method is probably the most helpful for me as it helped me to look back into the code thoroughly and understand the situation better.

  • Reading error messages - I'd say I've gotten used to reading the error messages a lot better throughout the exercise. However, I still think there is room for improvement.

  • Console.logging - I like to visualize the outcome, so I've used a lot of console.log. I am really confident with using console.log problem-solving method.

  • Googling - There are a few websites that I've got help from. It was usually helpful to find me the method of using a specific function.

  • Asking your peers for help - I don't think I have used this as my problem-solving skill.

  • Asking coaches for help - I usually like to do enough research and if I really do not understand a concept, then I would reach out for help. I am not the best person to use this problem-solving method.

  • Improving your process with reflection - I ask for help from my facilitators for this method as I am always on the lookout for improvements if necessary.


Reflect on a time you were reluctant to ask for help. Consider what made you reluctant to do so.

There was a time that I was reluctant to ask on Discord. While I was struggling with using 'map()' function, I thought I had a good understanding of what the function does and how to use it. After struggling for a while, I decided to ask on Discord for help, but before taking action, I had one last look into my code to see if I was missing out on a simple problem. Funny to say, I managed to find the error on my own and ended up not asking about the question.

Overall, the things that make me reluctant to ask for help are because I keep thinking that it's a simple problem and it's something that I can fix. Usually, that may end up me falling into a long loop of repeated stress, so I try my best to not fall into one.
